# Лабораторная работа №1: Защита данных с помощью AES-256

---

## 📋 Постановка задачи

Реализовать защиту данных пользовательских папок и файлов (включая все вложенные подпапки) путем шифрования. Для доступа к данным в исходной папке необходимо выполнить дешифрование.

### Основные требования:
- **Интерфейс приложения** — консольный
- **Язык реализации** — C++

## 🔐 Выбор библиотек для шифрования

В качестве основной библиотеки для реализации шифрования используется **OpenSSL** с реализацией алгоритма **AES-256**.

| Библиотека | Описание |
|------------|----------|
| OpenSSL (EVP API) | Мощная криптографическая библиотека, реализующая AES-256 в режиме CBC |
| Qt Core | Кроссплатформенный фреймворк для работы с файловой системой, хэшированием и логированием |

## 📂 Работа с файловой системой

Для рекурсивного обхода папок и шифрования всех вложенных файлов используется библиотека **Qt**, которая предоставляет богатый инструментарий для работы с директориями и файлами (`QDir`, `QFileInfo`, `QFile`).

## ⚙️ Требования к реализации

### 1. Архитектура приложения
- Реализован основной класс `CryptoManager`, обеспечивающий шифрование (encryption) и дешифрование (decryption)
- Класс реализован как **Singleton**
- Реализован класс `Logger` для ведения детальных логов операций (синглтон)

### 2. Входные данные
Приложение принимает на вход **три обязательных параметра**:

| Параметр | Описание | Пример |
|----------|----------|--------|
| **Путь к папке** | Полный или относительный путь к директории, содержимое которой будет обработано | `C:\Users\Документы\Secret` или `./test_folder` |
| **Пароль** | Строка для генерации ключа шифрования | `MySecurePassword123` |
| **Режим работы** | 1 — шифрование, 2 — дешифрование | `1` или `2` |

Все параметры вводятся пользователем последовательно в консоли.

### 3. Функциональность
- Рекурсивный обход всех подпапок
- Вычисление SHA-256 хэшей для контроля целостности (функционал заложен, вывод в консоль отключен)
- Проверка статуса файла (зашифрован/не зашифрован) по специальному маркеру
- **Защита от повторного шифрования/дешифрования**: программа пропускает файлы, уже находящиеся в целевом состоянии
- **Защита системных файлов**: программа автоматически пропускает файлы с атрибутами `SYSTEM`, `HIDDEN`, `DEVICE`, а также файлы, недоступные для записи
- **Детальное логирование**: все операции записываются в отдельные файлы (`logs_encrypt.txt`, `logs_decrypt.txt`, `logs_errors.txt`, `logs_info.txt`) с временными метками
- **Подведение статистики**: после выполнения операции выводится количество успешно обработанных, пропущенных файлов (с детализацией по системным) и ошибок

## 🔐 Программа для шифрования/дешифрования файлов с контролем целостности (AES-256)

Данная программа предназначена для рекурсивного шифрования и дешифрования всех файлов в указанной папке с использованием алгоритма AES-256 (режим CBC). Программа написана на C++ с использованием Qt Core и OpenSSL.

## 📌 Реализованное решение

### Общая идея
Пользователь вводит:
- **полный путь к папке** для обработки;
- **пароль** (используется для генерации ключа AES-256 и вектора инициализации);
- **режим работы**: **1** — шифрование, **2** — дешифрование.

Программа:
1. Проверяет существование указанной папки.
2. Рекурсивно обходит все файлы внутри неё с помощью `collectFilesInfo`.
3. **В режиме шифрования:**
   - проверяет, не является ли файл системным или защищенным (такие файлы пропускаются с занесением в лог);
   - проверяет наличие маркера шифрования; если файл уже зашифрован — пропускает его;
   - шифрует каждый файл алгоритмом AES-256 (CBC);
   - в начало каждого зашифрованного файла записывается специальный маркер (`0xEF 0xBE 0xAD 0xDE 0x01 0x02 0x03 0x04`) для идентификации зашифрованных файлов.
4. **В режиме дешифрования:**
   - проверяет, не является ли файл системным (пропускает);
   - проверяет наличие маркера с помощью `isFileEncrypted`;
   - дешифрует файлы, если они зашифрованы;
   - пропускает уже расшифрованные файлы.
5. По окончании работы выводит подробную статистику и сохраняет все действия в лог-файлы.

## 🏗 Архитектура ПО

### Основные сущности (классы)

| Класс/Структура | Назначение |
|-----------------|------------|
| `CryptoManager` | Синглтон, отвечающий за шифрование/дешифрование файлов с использованием AES-256. Содержит ключ и IV, полученные из пароля. |
| `Logger` | Синглтон для логирования операций в файлы. |
| `FileInfo` | Структура для хранения информации о файле: путь, относительный путь, размер, флаг `isEncrypted`. |
| Глобальные функции | `collectFilesInfo`, `calculateSHA256` — вспомогательные функции для работы с файловой системой и хэшированием. |

### Использованный паттерн: Singleton
Паттерн **Singleton** гарантирует, что у классов `CryptoManager` и `Logger` существует только один экземпляр на всё приложение. Это важно, потому что:
- ключи шифрования должны быть едины для всех операций;
- логи должны писаться в одни и те же файлы из любой точки программы.

## 🔧 Используемые технологии

### 1. Qt Core (фреймворк)
**Суть технологии:**
Qt — кроссплатформенный фреймворк для разработки на C++. В программе используются модули `QCoreApplication`, `QFile`, `QDir`, `QFileInfo`, `QCryptographicHash`, `QMutex`, `QDateTime`.

**Решаемые задачи:**
- Удобная работа с файловой системой (обход папок, проверка существования, получение относительных путей).
- Кроссплатформенное чтение/запись файлов.
- Встроенная поддержка SHA-256 через `QCryptographicHash`.
- Потокобезопасное логирование с помощью `QMutex`.
- Формирование временных меток для логов.

### 2. OpenSSL (криптографическая библиотека)
**Суть технологии:**
OpenSSL — мощная библиотека, реализующая SSL/TLS и различные криптографические алгоритмы, включая AES.

**Решаемые задачи:**
- Генерация ключа (SHA-256) и IV (SHA-1) из пароля.
- Шифрование и дешифрование данных по алгоритму AES-256 в режиме CBC.

**Использование EVP API:**
- `EVP_CIPHER_CTX_new()` — создание контекста.
- `EVP_EncryptInit_ex` / `EVP_DecryptInit_ex` — инициализация с ключом и IV.
- `EVP_EncryptUpdate` / `EVP_DecryptUpdate` — обработка блоков данных.
- `EVP_EncryptFinal_ex` / `EVP_DecryptFinal_ex` — финализация (добавление/удаление padding).

**Почему EVP API?**
EVP — высокоуровневый интерфейс, который позволяет легко менять алгоритмы без изменения логики, а также использует современные и безопасные функции OpenSSL.

## 🔐 Безопасная разработка

### 1. Обработка пароля
- Пароль не хранится в открытом виде — из него сразу генерируется ключ и IV.
- Используется SHA-256 для ключа (32 байта) и SHA-1 для IV (обрезается до 16 байт).
- Ключ и IV хранятся в `unsigned char` массивах и обнуляются с помощью `secureZero` после использования (в деструкторе и при инициализации).
- Пароль в памяти затирается (`assign(password.size(), '0')`) сразу после генерации ключа.

### 2. Проверка целостности
- В коде заложена функция `calculateSHA256` для вычисления хэшей, однако вывод в консоль отключен для улучшения пользовательского опыта. При необходимости её можно легко активировать.

### 3. Маркер зашифрованного файла
- В начало файла записывается уникальная сигнатура `ENCRYPTED_MARKER` (8 байт).
- Функция `isFileEncrypted` проверяет наличие этого маркера, что предотвращает повторное шифрование и позволяет быстро определить статус файла.

### 4. Защита системных файлов
- Функция `isProtectedSystemFile` проверяет атрибуты файла (Windows) и список защищенных имен.
- Файлы с атрибутами `SYSTEM`, `HIDDEN`, `DEVICE`, а также файлы, недоступные для записи, автоматически пропускаются.
- Это предотвращает повреждение важных системных файлов и обеспечивает безопасность ОС.

### 5. Логирование
- Все действия записываются в отдельные файлы: шифрование, дешифрование, ошибки, информация.
- Логи содержат временные метки и детализацию по каждому файлу.
- В конце каждой операции выводится итоговая статистика и записывается в лог.

### 6. Работа с памятью и файлами
- Буферы фиксированного размера (4096 байт) для чтения/записи.
- Временные файлы (`*.tmp`) создаются для безопасной обработки и удаляются в случае ошибки.
- Атомарность операций: при ошибке оригинальный файл не повреждается.

## ❓ Инструкция для пользователя по Qt Creator

### 1. Подготовка к использованию
Чек-лист:
1. Установить операционную систему Windows 10 (тип системы x64).
2. Скачать и настроить программное обеспечение Qt 5.15.2 (MinGW_64).

### 2. Клонирование проекта и настройка запуска
1. Скачайте этот проект любым удобным способом.
2. Откройте файл `CMakeLists.txt` (в папке `Lab_1`) и замените путь в переменной `OPENSSL_ROOT_DIR` на путь к директории OpenSSL на Вашем компьютере.
3. Откройте приложение `Qt Creator` -> `Open Project` -> откройте в скаченом проекте папку `Lab_1` и выберите файл `CMakeLists.txt`.
4. Перейдите в режим "Проекты" (`Ctrl + 5`) -> раздел "Сборка и запуск" -> в настройках запуска поставьте галочку "Запускать в терминале".
5. Нажмите `Запустить` или `Ctrl + R`.

## Как пользоваться консольным приложением?

Обратите внимание: повторное шифрование/дешифрование в системе не выполняется, поскольку это противоречит принятым мерам безопасности.

### Шифрование файлов
1. Введите **полный путь** к папке, которую вы хотите зашифровать (например, `C:\Users\YourName\Documents\Secret`).
2. Введите пароль.
3. Введите `1` -> `Enter`.

### Дешифрование файлов
Аналогично шифрованию, только на третьем этапе введите `2` -> `Enter`.

### Логирование
После выполнения операции в папке `build` с программой появятся файлы:
- `logs_encrypt.txt` — лог шифрования
- `logs_decrypt.txt` — лог дешифрования
- `logs_errors.txt` — лог ошибок
- `logs_info.txt` — информационные сообщения и статистика

## 👾 Процесс тестирования

Тестирование программы проводилось по следующим сценариям, охватывающим как штатные режимы работы, так и некорректные действия пользователя.

### Case 1: Нормальное шифрование и дешифрование
Проверка корректности работы программы: файлы в указанной папке успешно шифруются с заданным паролем и полностью восстанавливаются при дешифровании тем же паролем.

### Case 2: Неправильный пароль при дешифровании
Проверка реакции программы на ввод неверного пароля: файлы не должны изменяться, а программа обязана выдать сообщение об ошибке.

### Case 3: Пустой пароль
Проверка обработки пустого пароля: программа должна отклонить его как недопустимый.

### Case 4: Несуществующая папка
Проверка поведения при указании несуществующего пути: программа должна сообщить об ошибке и завершить работу.

### Case 5: Неверный режим работы
Проверка ввода некорректного режима (например, `3` вместо `1` или `2`): программа должна выдать предупреждение о неверном выборе.

### Case 6: Защита системных файлов
Проверка, что файлы с атрибутами `SYSTEM` или `HIDDEN` не изменяются и попадают в статистику как "пропущенные (системные)".

Для каждого теста фиксировались:
- изменение содержимого файлов;
- наличие информативных сообщений об ошибках и логов;
- код возврата программы.

Тестирование проводилось на наборе файлов в папке [TestFiles](https://github.com/ProSecureCodeByDaniil/RSZI/tree/develop/TestFiles).
