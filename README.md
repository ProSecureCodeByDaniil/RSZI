## 📋 Постановка задачи

Реализовать защиту данных пользовательских папок и файлов (включая все вложенные подпапки) путем шифрования. Для доступа к данным в исходной папке необходимо выполнить дешифрование.

### Основные требования:
- **Интерфейс приложения** — консольный
- **Язык реализации** — C++

## 🔐 Выбор библиотек для шифрования

В качестве основной библиотеки для реализации шифрования используется **OpenSSL** с реализацией алгоритма **AES-256**.

| Библиотека | Описание |
|------------|----------|
| OpenSSL (EVP API) | Мощная криптографическая библиотека, реализующая AES-256 в режиме CBC |
| Qt Core | Кроссплатформенный фреймворк для работы с файловой системой и хэшированием |

## 📂 Работа с файловой системой

Для рекурсивного обхода папок и шифрования всех вложенных файлов используется библиотека **Qt**, которая предоставляет богатый инструментарий для работы с директориями и файлами (`QDir`, `QFileInfo`, `QFile`).

## ⚙️ Требования к реализации

### 1. Архитектура приложения
- Реализован основной класс `CryptoManager`, обеспечивающий шифрование (encryption) и дешифрование (decryption)
- Класс реализован как **Singleton**

### 2. Входные данные
Приложение принимает на вход:
- **Название папки** для шифрования/дешифрования (поиск выполняется автоматически)
- **Пароль** для шифрования
- **Режим работы**: 1 — шифрование, 2 — дешифрование

### 3. Функциональность
- Рекурсивный обход всех подпапок
- Вычисление и вывод SHA-256 хэшей для контроля целостности
- Проверка статуса файла (зашифрован/не зашифрован) по специальному маркеру

## 🔐 Программа для шифрования/дешифрования файлов с контролем целостности (AES-256)

Данная программа предназначена для рекурсивного шифрования и дешифрования всех файлов в указанной папке с использованием алгоритма AES-256 (режим CBC). Перед шифрованием программа вычисляет SHA-256 хэши исходных файлов, а после дешифрования — повторно вычисляет хэши для проверки целостности данных. Программа написана на C++ с использованием Qt Core и OpenSSL.

## 📌 Реализованное решение

### Общая идея
Пользователь вводит:
- название папки (поиск выполняется вверх/вниз от текущей директории на глубину до 5 уровней);
- пароль (используется для генерации ключа AES-256 и вектора инициализации);
- режим работы: **1** — шифрование, **2** — дешифрование.

Программа:
1. Находит указанную папку с помощью функции `findFolder`.
2. Рекурсивно обходит все файлы внутри неё с помощью `collectFilesInfo`.
3. **В режиме шифрования:**
   - вычисляет и выводит SHA-256 хэши исходных файлов;
   - шифрует каждый файл алгоритмом AES-256 (CBC);
   - в начало каждого зашифрованного файла записывается специальный маркер (`0xEF 0xBE 0xAD 0xDE 0x01 0x02 0x03 0x04`) для идентификации зашифрованных файлов.
4. **В режиме дешифрования:**
   - проверяет наличие маркера с помощью `isFileEncrypted`;
   - дешифрует файлы;
   - повторно вычисляет SHA-256 хэши для проверки целостности.

## 🏗 Архитектура ПО

### Основные сущности (классы)

| Класс/Структура | Назначение |
|-----------------|------------|
| `CryptoManager` | Синглтон, отвечающий за шифрование/дешифрование файлов с использованием AES-256. Содержит ключ и IV, полученные из пароля. |
| `FileInfo` | Структура для хранения информации о файле: путь, относительный путь, размер, флаг `isEncrypted`. |
| Глобальные функции | `collectFilesInfo`, `findFolder`, `calculateSHA256`, `calculateAndPrintHashes` — вспомогательные функции для работы с файловой системой, поиском папок и хэшированием. |

### Использованный паттерн: Singleton
Паттерн **Singleton** гарантирует, что у класса `CryptoManager` существует только один экземпляр на всё приложение. Это важно, потому что:
- ключи шифрования должны быть едины для всех операций;
- нет необходимости создавать несколько менеджеров шифрования.

Архитектура паттерна в программе:
- приватный статический указатель `instance`;
- приватный конструктор;
- публичный статический метод `getInstance()`;
- метод `destroyInstance()` для явного удаления экземпляра;
- запрет копирования и присваивания.

## 🔧 Используемые технологии

### 1. Qt Core (фреймворк)
**Суть технологии:**
Qt — кроссплатформенный фреймворк для разработки на C++. В программе используются модули `QCoreApplication`, `QFile`, `QDir`, `QFileInfo`, `QCryptographicHash`.

**Решаемые задачи:**
- Удобная работа с файловой системой (обход папок, проверка существования, получение относительных путей).
- Кроссплатформенное чтение/запись файлов.
- Встроенная поддержка SHA-256 через `QCryptographicHash`.

**Шаги использования:**
- Подключение заголовочных файлов (`#include <QCoreApplication>` и др.).
- Создание объектов `QDir`, `QFileInfo` для навигации.
- Использование `QCryptographicHash::Sha256` для вычисления хэшей.
- Управление файлами через `QFile::remove`, `QFile::rename`.

### 2. OpenSSL (криптографическая библиотека)
**Суть технологии:**
OpenSSL — мощная библиотека, реализующая SSL/TLS и различные криптографические алгоритмы, включая AES.

**Решаемые задачи:**
- Генерация ключа (SHA-256) и IV (SHA-1) из пароля.
- Шифрование и дешифрование данных по алгоритму AES-256 в режиме CBC.

**Шаги использования:**
- Подключение заголовков: `<openssl/aes.h>`, `<openssl/evp.h>`, `<openssl/sha.h>`.
- Использование **EVP API** (рекомендуемый интерфейс OpenSSL):
  - `EVP_CIPHER_CTX_new()` — создание контекста.
  - `EVP_EncryptInit_ex` / `EVP_DecryptInit_ex` — инициализация с ключом и IV.
  - `EVP_EncryptUpdate` / `EVP_DecryptUpdate` — обработка блоков данных.
  - `EVP_EncryptFinal_ex` / `EVP_DecryptFinal_ex` — финализация (добавление/удаление padding).
  - Освобождение ресурсов: `EVP_CIPHER_CTX_free`.

**Почему EVP API?**
EVP — высокоуровневый интерфейс, который позволяет легко менять алгоритмы без изменения логики, а также использует современные и безопасные функции OpenSSL.

## 🔐 Безопасная разработка

### 1. Обработка пароля
- Пароль не хранится в открытом виде — из него сразу генерируется ключ и IV.
- Используется SHA-256 для ключа (32 байта) и SHA-1 для IV (обрезается до 16 байт).
- Ключ и IV хранятся в `unsigned char` массивах и обнуляются с помощью `secureZero` после использования (в деструкторе и при инициализации).
- Пароль в памяти затирается после генерации ключа.

### 2. Проверка целостности
- До шифрования вычисляются хэши SHA-256 — они служат эталоном и выводятся на экран.
- После дешифрования хэши вычисляются повторно — пользователь может сравнить их с исходными.
- Это позволяет обнаружить:
  - повреждение данных;
  - неверный пароль (при дешифровании получится "мусор", хэши не совпадут).

### 3. Маркер зашифрованного файла
- В начало файла записывается уникальная сигнатура `ENCRYPTED_MARKER` (8 байт).
- Функция `isFileEncrypted` проверяет наличие этого маркера, что предотвращает повторное шифрование уже зашифрованных файлов и позволяет быстро определить статус файла.

### 4. Работа с памятью
- Буферы фиксированного размера (4096 байт) для чтения/записи.
- Проверка количества реально прочитанных/записанных байт.
- Временные файлы (`*.tmp`) создаются для безопасной обработки и удаляются в случае ошибки.
- Использование `std::vector` для автоматического управления буферами.

### 5. Обработка ошибок
- Проверка всех операций с файлами и криптографических вызовов.
- Информативные сообщения об ошибках в консоли.
- Атомарность операций: при ошибке оригинальный файл не повреждается.

## 📝 Заключение

Программа демонстрирует:
- грамотное использование Qt для работы с файловой системой;
- применение OpenSSL (EVP API) для криптографических операций;
- реализацию паттерна Singleton с безопасным управлением памятью;
- базовые принципы безопасной разработки (обработка паролей, проверка целостности, временные файлы, затирание чувствительных данных).

При необходимости программу можно расширить: добавить поддержку других режимов шифрования, многопоточность для ускорения обработки большого количества файлов или графический интерфейс.

## ❓ Инструкция для пользователя по Qt Creator

### 1. Подготовка к использованию
Чек лист:
1. Установить операционную систему Windows 10 (тип системы x64).
2. Скачать и настроить программное обеспечение Qt 5.15.2 (MinGW_64).
3. Установить статическую библиотеку OpenSSL ([версия 3.6.1](https://disk.yandex.ru/d/xqiiqt2AfRhBUw)).

### 2. Клонирование проекта и настройка запуска
1. Скачивайте этот проект любым удобным Вам способом.
2. Откройте теперь `CMakeLists.txt` (в папке `Lab_1`) и замените на свой путь к директории OpenSSL на Вашем компьютере.
3. Открываем приложение `Qt Creator` -> `Open Project` -> Открываем папку `Lab_1` -> Выбираем `CMakeLists.txt`.
4. Перейдите в режим проекты (`Ctrl + 5`) -> Сборка и запуск (выбираем запустить) -> Запускать в терминале (ставим галочку).
5. Нажмите на `Запустить` или `Ctrl + R`.

## Как пользоваться консольным приложением?
Обратите внимание: повторное шифрование/дешифрование в системе не выполняется, поскольку это противоречит принятым мерам безопасности.
А также рекомендую разместить рядом с проектом папку которую вы хотите зашифровать — это обеспечит быстрый доступ и повысит эффективность работы!

### Шифрование файлов
1. Введите название папки которую вы хотите шифровать.
2. Введите свой придуманный сложный пароль.
3. Введите `1` -> `Enter`.

### Дешифрование файлов
Аналогично шифрованию, только на третьем этапе введите `2` -> `Enter`.
